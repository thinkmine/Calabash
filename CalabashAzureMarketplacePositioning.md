
# Calabash
## Azure Marketplace Product Positioning Document

### Product Name
**Calabash – Agent Infrastructure Platform for Enterprise AI**

---

# Executive Summary

Calabash is an **Agent Infrastructure Platform (AIP)** designed to help organizations operationalize AI agents securely within enterprise environments.

While many AI platforms provide access to models, enterprises struggle to connect those models safely to real business systems. Calabash solves this challenge by acting as the **runtime layer that connects AI agents to enterprise tools, data, and services**.

Calabash enables organizations to:

* Build AI agents that can take real actions
* Connect agents to enterprise systems and databases
* Govern and audit AI-driven operations
* Maintain data sovereignty through private deployments
* Accelerate development of AI-native applications

Deployed as a **virtual machine in the customer's Azure environment**, Calabash provides enterprises with full control of their infrastructure, security boundaries, and data.

---

# The Problem Enterprises Face

Enterprises adopting generative AI often discover that:

* LLMs cannot directly interact with enterprise systems
* Integrating AI with legacy databases is complex
* Governance and auditing are missing from many AI tools
* Data sovereignty requirements limit SaaS adoption
* Developers must build agent infrastructure from scratch

Without an infrastructure layer, organizations struggle to move from **AI experiments to operational AI systems**.

---

# Calabash Solution

Calabash provides the **agent runtime, integration framework, and governance layer** required to operationalize AI agents inside enterprise environments.

The platform allows organizations to expose enterprise systems as **AI tools**, enabling agents to safely interact with business systems.

Core capabilities include:

* AI agent hosting and orchestration
* Tool and skill execution framework
* Enterprise database integration
* Model integration with Azure AI services
* Secure execution environment for agent workflows

---

# Key Capabilities

## Agent Runtime Server

Calabash runs AI agents in a controlled runtime environment where agents can:

* reason
* plan actions
* call tools
* interact with enterprise systems

The runtime coordinates model calls, tool execution, and workflow orchestration.

---

## Tool and Skill Framework

Calabash enables developers to create reusable **agent tools (skills)** implemented in multiple languages including:

* .NET plugins
* Python scripts
* JavaScript modules
* REST APIs

These tools can be safely invoked by agents during execution.

---

## Enterprise Data Connectivity

Calabash can expose enterprise databases as AI-accessible endpoints including:

* SQL Server tables
* views
* stored procedures

These endpoints can be used directly by AI agents as structured tools.

---

## Azure AI Model Integration

Calabash integrates with **Azure AI Foundry deployments**, allowing agents to use models deployed in the customer's Azure subscription.

Benefits include:

* secure model access
* enterprise identity integration
* controlled model usage

---

## Secure Enterprise Deployment

Calabash runs inside the customer's Azure tenant as a virtual machine.

This provides:

* full infrastructure control
* enterprise network integration
* private data access
* compliance with regulatory requirements
